## 🚀 启动确认任务（每次新会话自动执行）

### 真正的阅读流程

1. **读取 Skill 文件**（使用 `read` 工具）
   - `skills/execution-standards/SKILL.md` - 重点：遇阻停顿原则、PowerShell 限制、长时间任务汇报
   - `skills/web-search/SKILL.md` - 重点：搜索流程、结果验证
   - `skills/multi-model-async/SKILL.md` - 重点：多模型分配、API Key 配置

2. **理解并内化关键规范**
   - 不是形式上的"已读"，而是真正理解规范要求
   - 思考：这些规范如何影响我接下来的工作？

3. **在回复中体现理解**
   - 不只是说"准备就绪"
   - 要体现对关键规范的理解

4. **在后续工作中应用**
   - 遇阻时先查 Skill
   - 不用 PowerShell 除非授权
   - 长时间任务每 5 分钟汇报

## 💡 说明

**为什么用 Heartbeat？**
- OpenClaw 每次启动时会触发一次 heartbeat
- 利用这个机制自动完成启动确认
- **真正的目的**：刷新对规范的记忆，确保后续工作符合标准

**执行时机**：
- 新会话启动时
- Gateway 重启后
- 长时间无活动后的重新连接
